Asset Types

The four visual types and when to use each one.

The Goal

This is what a finished annotated script looks like. Your job is to take a raw script and turn it into this — every sentence paired with a YouTube clip or image reference so the editor can assemble the video without guessing:

HOOK

It's March 2008 - and Bear Stearns is collapsing overnight [https://www.youtube.com/watch?v=DcyaQBCGM28, 0:49 - 0:52]. Lehman Brothers soon follows [https://www.youtube.com/watch?v=-MKHah7QQpY, 0:28 - 0:31], defaulting on billions of mortgage-backed securities. [pic 1]

The entire financial system is imploding in real time. [https://www.youtube.com/watch?v=l1DBDONn9wQ, 0:02 - 0:06] and when every major bank is on the brink of collapse [https://www.youtube.com/watch?v=nP1FxdmMZpI, 14:02 - 14:04], only one institution gets the call to save the system: JPMorgan Chase

Fast forward to 2023 - and history is repeating itself. Silicon Valley Bank fails [pic 2] in 48 hours [https://www.youtube.com/watch?v=YDwDMF_f5rM, 0:20 - 0:23]. First Republic Bank follows right behind it. And once again, the government turns to the same bank.

So why is JPMorgan Chase always the last bank standing [https://www.youtube.com/watch?v=_zrRznfEjK4, 0:38 - 0:40]? And how did it become the most powerful financial institution in America? Today, we break down the full story.

The rest of this page explains each asset type and how to format them.

03B-Roll Video

Video footage that plays between host-on-camera segments. The B-roll plays on screen while Ben's voiceover narrates over it. This is the primary visual type, making up ~60% of screen time.

When to Use B-Roll

  • Showing a person in action or speaking (interview footage, news coverage, conference keynotes)
  • Demonstrating a location, building, or environment
  • Illustrating a concept, process, or activity
  • Movie or TV clips that visually match what's being discussed (e.g., Wolf of Wall Street trading floor scenes when talking about Wall Street culture)
  • Filling gaps between still images to maintain visual variety and pacing
  • The original audio from the clip is not relevant to the narrative (most use cases)

Rules

  • 3-10 seconds per clip (ideal: 4-7 seconds). No isolated clips under 3 seconds.
  • Must match the era discussed in the script — no 2024 footage for 1986 events
  • Must pass the relevance test: if watched on mute, could you guess the topic?
  • Always pick the more interesting clip when two exist
  • Never reuse the same clip for two different script lines (unless revisiting a topic)
  • Stock footage is a last resort (under 10% of total)

How to Annotate

Visual Distinction

B-roll with no "WITH AUDIO" marker means just the video plays while Ben narrates over it. This is the default — the footage is visual-only. It typically appears between host-on-camera segments.

Correct Format

Always use this format — the full YouTube URL, followed by the exact start and end timestamps separated by a dash. Each clip goes in its own set of brackets:

[https://www.youtube.com/watch?v=QZ85VPgCOCM, 0:18 - 0:29] [https://www.youtube.com/watch?v=nbs_s6_L_ao, 22:10 - 22:14]

Common Mistakes

These will get flagged and sent back for revision:

Wrong — Do Not Do These [Goldman Sachs CEO Interview, 0:18 - 0:29] ← No titles, raw URLs only [https://youtube.com/watch?v=abc, around 3 min] ← No vague timestamps [https://youtube.com/watch?v=abc] ← Never omit timestamps

04Still Images

A static image displayed on screen, shown for 2-6 seconds. Still images make up ~25% of screen time.

Image Types (In Order of Preference)

  1. News article screenshots with key phrases highlighted — the signature rareliquid look. The editor will handle the highlighting, so just find and screenshot the article. Always find the original article, not a tweet about it.
  2. Official documents and filings — SEC filings, court docs, press releases, earnings reports. Crop to the relevant section.
  3. Data charts and graphs — from reputable sources (Bloomberg, WSJ, company filings). Must be legible at 1080p. These can also be remade with Claude Code and served as an HTML file for a cleaner look.
  4. High-resolution photographs — professional headshots, event photos, campus/building shots. For historical subjects, search "[name] young" for age-appropriate photos.
  5. Social media screenshots — only when directly relevant and from notable accounts.

Quality Requirements

  • Minimum resolution: 1280x720 (ideally 1920x1080 or higher)
  • No watermarks, no compression artifacts
  • Crop to relevant content — no browser chrome, tabs, or bookmarks

Naming Convention

Name every image sequentially: 1.png, 2.png, 3.png. Upload the entire folder to Slack as a batch.

05B-Roll with Audio

Same as B-roll, but the original audio plays instead of Ben's voiceover. Very rare: ~2% of total video, only 3-5 uses per 15-minute video.

When to Use (Sparingly)

  • A CEO or public figure makes a critical statement that is more powerful heard in their own voice
  • A news anchor is breaking a major story and their delivery adds drama and credibility
  • An interview subject makes a quote that deserves to be heard directly, not summarized
  • A moment of genuine emotion or emphasis that loses impact if paraphrased

How to Annotate

06Custom Visuals (HTML)

Custom-built HTML files rendered as graphics. Only 1-3 per entire video. Do not overuse.

When to Flag for a Custom Visual

  • Career timelines (e.g., Irving Trust → Drexel → Bear Stearns → Goldman)
  • Financial data: revenue, profit margins, stock price comparisons
  • Comparison tables: two candidates side by side, company metrics vs. peers
  • Company splits, mergers, organizational changes
  • Key statistics or quotes that deserve visual emphasis

How to Build These

If there's a chart, data breakdown, or document-style graphic that would look good on screen, use an LLM (Claude Opus 4.6 recommended) to generate an HTML file for it. Send me the HTML file and I can style it to match our theme. This works great for timelines, financial breakdowns, comparison tables — basically anything where a clean visual beats a random screenshot, or when there are no relevant pictures.

See 5:05 (JPM splitting up) and 10:05 (JPM revenue/income data) in the JPM documentary for examples of what these look like in the final video.

LLMs, especially Claude, are surprisingly good at this and it usually only takes a prompt or two. Just make sure to have it cite sources if you're not 100% sure about the data, since they can hallucinate. Ideally, instruct it to pull from online sources so everything is verifiable.

Example Prompts & Results

Here's what the process looks like in practice. Each example shows the prompt on the left and the generated output on the right:

Data Dashboard

Prompt: "give me pictures graphics tables whatever that show how jp morgan dominates commercial and investment banking" — Claude created a full dashboard with market share charts, bank rankings, M&A league table, and revenue breakdown.

Company Split Graphic

Prompt: "create me a graphic that shows jp morgan dividing into jp morgan & co commercial bank and morgan stanley investment bank" — then refined with "make it without words much more simple because I'll be voicing over in a youtube channel". Usually takes just one or two follow-ups.

Revenue Chart with Live Data

Prompt: "recreate this image but start it from 2007 and include jp morgan's revenue in 2024 and 2025" — Claude searched the web for the latest figures and generated a clean bar chart. A quick "no decimal points" follow-up polished it.

How to Annotate

Reference custom visuals by filename, just like still images:

Format[15.html]

Quality Checklist

  • All text legible at 1080p
  • Data sourced and cited
  • Self-contained HTML file (no external dependencies)
  • 16:9 aspect ratio
